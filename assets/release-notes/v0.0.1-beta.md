# Release v0.0.1-beta

## What's New

- **Two-phase workflow** — `restrail init` generates editable `.request` files and an `execution.plan` from your OpenAPI spec; `restrail run` executes them against a live API. Phases can be combined with `restrail -i init.json run` for one-shot CI use.
- **Profile system** — five built-in profiles cover the most common REST API shapes: `strict` (PUT by ID), `ddd` (PUT on collection), `patchwork` (PATCH), `appendonly` (POST + GET only), `readonly` (GET only). Custom profiles can be loaded from disk.
- **Profile discovery** — `restrail discover-profile` auto-detects the best-matching profile for a given OpenAPI spec.
- **Foreign key dependency detection** — automatically detects FK relationships between domains by scanning POST body schemas, topologically sorts execution order, and injects resolved IDs at runtime.
- **Authentication** — auto-detected from `components.securitySchemes`: Basic, Bearer token, OAuth2 (client credentials, password grant, authorization code), OIDC, or unauthenticated.
- **Optimistic locking support** — when enabled, `version`/`v`/`ver` fields are extracted from GET/POST responses and injected into subsequent PUT/PATCH request bodies.
- **Blocked domains** — individual domains can be excluded from a run via `blocked_domains` in the run config or the `RESTRAIL_BLOCKED_DOMAINS` environment variable. Blocked domains appear as skipped in reports rather than failed.
- **Report generation** — results output as JSON, Markdown, and/or HTML via `--format`. Multiple formats can be requested in one run.
- **Testdata factory** — generates random, schema-conforming request bodies using a fixed seed (42) for reproducible runs across environments.
- **Zero external dependencies** — built entirely on the Go standard library.
